CamController::CamController ( int cameraIndex = 0,
int width = 640,
int height = 480 )

Constructs a CamController and pre-allocates internal frame buffers.

Sets all tuneable parameters to their defaults and registers the built-in "red" colour (hue bands 0–10 and 170–180) so the controller is usable immediately after construction without any further colour configuration.

The combined mask and temporary mask are allocated once here to avoid per-frame heap allocations inside the hot capture path.

Parameters
cameraIndexV4L2 / OpenCV device index (ignored when rpicam-vid is used). Default: 0.
widthDesired capture width in pixels. Default: 640.
heightDesired capture height in pixels. Default: 480.

Registers a new named colour for tracking.

Each colour is described by two HSV bands — a lower-hue band and an upper-hue band — which are OR-combined into the frame mask each cycle. This two-band design naturally handles red, which wraps around the HSV hue circle.

If name already exists its definition is updated in-place (no slot is consumed). If the table is full and name is new the function prints an error and returns false.

Parameters
nameUnique identifier for the colour (e.g. "orange").
lHueMinLower band: minimum hue value [0, 179].
lHueMaxLower band: maximum hue value [0, 179].
lSatMinLower band: minimum saturation [0, 255].
lValMinLower band: minimum value (brightness) [0, 255].
uHueMinUpper band: minimum hue value [0, 179].
uHueMaxUpper band: maximum hue value [0, 179].
uSatMinUpper band: minimum saturation [0, 255].
uValMinUpper band: minimum value (brightness) [0, 255].
Returns
true if the colour was added or updated successfully, false if MAX_TRACKED_COLORS would be exceeded.

◆ analyzeColorObjects()

CamController::Direction CamController::analyzeColorObjects ( )
private

Selects the primary target according to the active TrackingStrategy and computes the resulting Direction.

If no objects are detected, returns a Direction with command = "NO TARGET" and zeroed angle / distance fields.

Otherwise the primary target is chosen as follows:

  • LARGEST — object with the greatest contour area.
  • CLOSEST_TO_CENTER — object whose centre X is nearest the frame centre.
  • LEFTMOST — object with the smallest centre X coordinate.
  • LOWEST — object with the largest centre Y coordinate (closest to the bottom of the frame, i.e. nearest the robot).

The bearing angle is mapped linearly from the target's horizontal position: centre → 0°, full right → +90°, full left → −90°. When |angle| ≤ deadOnThreshold the command is "FORWARD", otherwise it is "LEFT N degrees" or "RIGHT N degrees", both suffixed with the matched colour name.

Returns
Populated Direction struct.

◆ captureFrame()

bool CamController::captureFrame ( )

Captures one frame from the camera and runs the full detection pipeline on it.

RPi path: reads exactly one YUV 4:2:0 frame from the rpicam-vid pipe, converts it to BGR, and optionally flips it 180°. The frame is stored under frameMutex_ so the streaming thread can access it safely.

Generic path: grabs from the OpenCV VideoCapture and flips if requested.

After storing the frame, detectColorPixels() and findColorObjects() are called to populate detectedObjects_ for the current cycle.

Returns
true if a frame was successfully read, false on stream error or empty frame.

◆ detectColorPixels()

void CamController::detectColorPixels ( )
private

Converts the current BGR frame to HSV and builds a combined colour mask for all registered colours.

Performs a single cv::cvtColor (BGR → HSV), then iterates over every registered ColorRange. For each colour both the lower and upper HSV bands are thresholded with cv::inRange into tmpMask_, which is then OR-merged into combinedMask_. Using pre-allocated member mats avoids heap allocations on the hot path.

Called internally by captureFrame().

◆ drawVisualization()

void CamController::drawVisualization ( cv::Mat & frame)
private

Renders all detection overlays onto frame in-place.

Draws the following elements:

  • Collection zone guide lines — two vertical purple lines at collectXMin_ and collectXMax_ (normalised X), labelled "COLLECT ZONE".
  • Per-object markers — a circle at the centroid and a bounding rectangle for every detected object. The primary target (selected by the current strategy) is drawn in green; all others in grey. Each is labelled with its colour name, index, and "(TARGET)" tag where applicable.
  • Frame-centre dot — blue filled circle at (cols/2, rows/2).
  • Target line — yellow line from the frame centre to the primary target centroid.
  • Alignment highlight — when the primary target's normalised X falls within the collection zone a semi-transparent green rectangle is overlaid and "ALIGNED" is printed.
Parameters
frameBGR image to draw on (modified in-place).

void CamController::enableStreaming ( bool enable,
int port = 8080 )

Enables or disables the background MJPEG streaming server.

Enabling: stores the port, sets streamingEnabled to true, and launches a detached std::thread running streamingLoop(). A message with the stream URL is printed to stdout.

Disabling: sets streamingEnabled to false (which causes streamingLoop() to exit its inner loops on the next iteration) and joins the thread.

Calling enableStreaming(true) while already streaming, or enableStreaming(false) while not streaming, is a no-op.

Parameters
enabletrue to start streaming, false to stop.

◆ findColorObjects()

std::vector< CamController::RedObject > CamController::findColorObjects ( )
private

Finds and filters colour blobs in the current combined mask.

Runs cv::findContours once on combinedMask_ (RETR_EXTERNAL, no mask copy needed). For each contour whose area exceeds minArea_:

  1. Centroid is computed from image moments (falls back to bounding-box centre when the moment is zero).
  2. Elevated-object filter rejects any blob that is both large (area > elevatedAreaThresh_) and sits in the top portion of the frame (normalised Y < elevatedYThresh_). With the camera ~7 cm off the ground, real cups always appear low in the frame; large blobs near the top are assumed to be elevated furniture.
  3. Colour identification samples the HSV pixel at the centroid and matches it against each registered ColorRange. The first match wins; unmatched blobs are labelled "unknown".

Called internally by captureFrame().

Returns
Vector of RedObject structs, one per accepted blob.

bool CamController::initialize ( )

Opens the camera and starts the rpicam-vid capture pipeline.

Launches rpicam-vid as a child process via popen(), requesting raw YUV 4:2:0 output on stdout at the resolution configured in the constructor. If the pipe cannot be opened the function prints an error and returns false; otherwise it sets the useRpiCam flag and returns true.

Note
This method is specific to Raspberry Pi. When porting to a platform without rpicam-vid, replace the body with an cv::VideoCapture open call and clear the useRpiCam flag.
Returns
true if rpicam-vid started successfully, false otherwise.

◆ streamingLoop()

void CamController::streamingLoop ( )
private

Background thread body for the single-client MJPEG streaming server.

Creates a TCP socket, binds to streamPort, and enters an accept loop. For each connected client:

  1. Sends the HTTP multipart response header.
  2. Enters a frame loop that grabs the latest visualised frame (under frameMutex), JPEG-encodes it at quality 80, wraps it in a MIME boundary part, and sends it.
  3. Sleeps ~66 ms between frames to target ~15 FPS.
  4. Exits the frame loop and closes the client socket when send() fails (client disconnected) or streamingEnabled becomes false.

Only one client is served at a time (listen(srv, 1)). The server socket is closed when streamingEnabled becomes false and the accept loop exits.

Note
Called exclusively by the thread launched in enableStreaming(). Do not call directly.
